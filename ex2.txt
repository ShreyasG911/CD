📘 2. Theory (Simple Explanation)
🔹 What is LEX?
LEX is a tool used to generate a Lexical Analyzer.
It:
•	Takes regular expressions as input
•	Generates a C program (lex.yy.c)
•	That program scans input and performs actions

💻 3. LEX Program (count.l)
Save this as: count.l
%{
#include <stdio.h>

int lines = 0;
int words = 0;
int spaces = 0;
int tabs = 0;
int characters = 0;
%}

%%
\n          { lines++; characters++; }
[ ]         { spaces++; characters++; }
\t          { tabs++; characters++; }
[a-zA-Z0-9]+ { words++; characters += yyleng; }
.           { characters++; }
%%

int main()
{
    printf("Enter the text (Press Ctrl+D to stop):\n");
    yylex();

    printf("\n----- COUNT RESULT -----\n");
    printf("Lines: %d\n", lines);
    printf("Words: %d\n", words);
    printf("Spaces: %d\n", spaces);
    printf("Tabs: %d\n", tabs);
    printf("Characters: %d\n", characters);

    return 0;
}

int yywrap()
{
    return 1;
}

🎓 Viva Questions
❓ What is yylex()?
Function that starts lexical analysis.
❓ What is yywrap()?
Function called at end of input.
❓ What is yytext?
Stores matched lexeme.
❓ What is yyleng?
Length of matched word.
❓ Difference between Lex and Compiler?
Lex generates lexical analyzer only.
